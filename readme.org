#+title: miniflux.el — Miniflux RSS client for Emacs (via elfeed)
#+author: bladrome
#+language: en

* Overview

=miniflux.el= is an Emacs package that turns [[https://miniflux.app][Miniflux]] into a first-class
RSS reader inside Emacs.  It fetches entries from your Miniflux server
and stores them in [[https://github.com/skeeto/elfeed][elfeed]]'s local database, so you get the full
elfeed UI (search, filtering, entry display) with bidirectional sync
of read/unread and starred status.

** Key features

- *Full elfeed integration* — entries live in elfeed's database and use
  its familiar search-mode interface.
- *Bidirectional sync* — read/unread and star/bookmark states are
  pushed back to the Miniflux server whenever you change them in
  elfeed.
- *API token or HTTP Basic Auth* — supports both authentication methods
  natively.
- *Category tags* — Miniflux categories are automatically imported as
  elfeed tags.
- *Small dependency surface* — only relies on built-in =url.el= and
  =elfeed=.

* Installation

** straight.el (recommended)

#+begin_src emacs-lisp
(straight-use-package
 '(miniflux :type git :host github :repo "bladrome/miniflux.el"))
#+end_src

** use-package + straight

#+begin_src emacs-lisp
(use-package miniflux
  :straight (miniflux :type git :host github :repo "bladrome/miniflux.el")
  :config
  (setq miniflux-server "https://your-miniflux.example.com")
  (setq miniflux-token "your-api-token"))
#+end_src

** Manual

Add the cloned repository directory to your =load-path= and require the package:

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/miniflux.el/")
(require 'miniflux)
#+end_src

* Configuration

| Variable                         | Default                  | Description                                         |
|----------------------------------+--------------------------+-----------------------------------------------------|
| =miniflux-server=                  | ="https://example.com"=    | Base URL of your Miniflux instance                  |
| =miniflux-token=                   | =""=                       | API token; preferred over other auth sources        |
| =miniflux-username=                | =""=                       | Username for explicit HTTP Basic Auth               |
| =miniflux-password=                | =""=                       | Password for explicit HTTP Basic Auth               |
| =miniflux-sync-limit=              | =200=                      | Max entries fetched per sync page                   |
| =miniflux-sync-star-tag=           | =star=                     | elfeed tag used for starred/bookmarked entries      |
| =miniflux-sync-failed-tag=         | =miniflux-sync-failed=     | tag added when a local push to Miniflux fails       |
| =miniflux-sync-full-unread=        | =t=                        | fetch all unread pages up to the page limit         |
| =miniflux-sync-full-starred=       | =t=                        | fetch all starred pages up to the page limit        |
| =miniflux-sync-pages-limit=        | =20=                       | maximum pages fetched for unread/starred queries    |
| =miniflux-sync-incremental=        | =t=                        | fetch recent entries changed since last sync        |
| =miniflux-category-tag-prefix=     | =""=                       | optional prefix for managed category tags           |
| =miniflux-web-entry-path=          | ="/unread/entry/%d"=       | Miniflux web URL path format for an entry           |
| =miniflux-web-feed-path=           | ="/feeds/%d/entries"=      | Miniflux web URL path format for a feed             |

** Example (API token)

#+begin_src emacs-lisp
(setq miniflux-server "https://miniflux.mydomain.com")
(setq miniflux-token "my-secret-api-token")
#+end_src

** Example (username + password)

#+begin_src emacs-lisp
(setq miniflux-server "https://miniflux.mydomain.com")
(setq miniflux-username "admin")
(setq miniflux-password "s3cret")
#+end_src

** Example (auth-source)

When =miniflux-token= is empty and explicit username/password are unset,
=miniflux.el= looks up credentials for the host in =auth-source=.  Use a
login of =token=, =api-token=, or =miniflux-token= to send the secret as an
=X-Auth-Token= header:

#+begin_src authinfo
machine miniflux.mydomain.com login token password my-secret-api-token
#+end_src

Any other login is treated as the username for HTTP Basic Auth:

#+begin_src authinfo
machine miniflux.mydomain.com login admin password s3cret
#+end_src

* Usage

** Interactive commands

| Command                                  | Description                                           |
|------------------------------------------+-------------------------------------------------------|
| =M-x miniflux=                             | Sync from server and open elfeed search buffer        |
| =M-x miniflux-sync=                        | Sync only; does not open elfeed                       |
| =M-x miniflux-sync-async=                  | Start a non-blocking sync and return immediately     |
| =M-x miniflux-search-refresh=              | Sync and refresh elfeed search                        |
| =M-x miniflux-refresh-current-feed=        | Refresh the selected entry's feed on Miniflux         |
| =M-x miniflux-mark-current-feed-read=      | Mark the selected entry's feed as read, then sync     |
| =M-x miniflux-mark-current-category-read=  | Mark the selected entry's category as read, then sync |
| =M-x miniflux-open-entry-in-miniflux=      | Open the selected entry in the Miniflux web UI        |
| =M-x miniflux-open-feed-in-miniflux=       | Open the selected entry's feed in the Miniflux web UI |
| =M-x miniflux-show-unread=                 | Show unread entries in elfeed search                  |
| =M-x miniflux-show-starred=                | Show starred entries in elfeed search                 |
| =M-x miniflux-show-category=               | Show entries from a Miniflux category tag             |
| =M-x miniflux-show-feed=                   | Show entries from the selected entry's feed           |
| =M-x miniflux-show-sync-failed=            | Show entries whose local push failed                  |
| =M-x miniflux-retry-sync-failed=           | Retry pushing selected entries' local state           |
| =M-x miniflux-clear-sync-failed=           | Clear failure markers from selected entries           |

** Keybindings in elfeed-search-mode

| Key | Action                                                          |
|-----+-----------------------------------------------------------------|
| =G=   | Sync from Miniflux server and refresh (=miniflux-search-refresh=) |
| =M=   | Mark current feed as read on Miniflux and sync                  |
| =O=   | Open current entry in the Miniflux web UI                       |
| =R=   | Retry pushing selected failed local changes                     |

Standard elfeed keybindings continue to work and automatically sync
back to Miniflux:

| Key | Action                                              |
|-----+-----------------------------------------------------|
| =s=   | Mark selected entries as read (syncs to Miniflux)   |
| =u=   | Mark selected entries as unread (syncs to Miniflux) |
| =+=   | Add a tag (e.g. =+ star= to sync star to Miniflux)    |
| -=  | Remove a tag (e.g. =- star= to unstar on Miniflux)    |

** How sync works

=miniflux-sync= performs the following steps:

1. Fetches /recent/, /unread/, and /starred/ entries from the Miniflux
   API.  Unread and starred entries can be fetched across multiple
   pages using =miniflux-sync-full-unread=,
   =miniflux-sync-full-starred=, and =miniflux-sync-pages-limit=.
2. Converts them to elfeed entries and writes them into the local
   elfeed database.
3. Reconciles local elfeed read/unread and star tags to match the
   server-side state when the server-side batch is known to be complete.
4. Reconciles Miniflux category tags.  With the default empty
   =miniflux-category-tag-prefix=, categories are added as bare slug tags
   and stale category tags are not pruned.  Set a non-empty prefix such as
   ="miniflux-category-"= to manage category tags in that namespace while
   preserving unrelated user tags.
5. Stores Miniflux IDs, category information, and feed details in entry
   metadata where elfeed can preserve them locally.
6. Saves the elfeed database and records =miniflux-last-sync-time= for
   incremental recent-entry sync.

Sync is pull-only: the Miniflux server is the source of truth during
=miniflux-sync=.  Local changes are pushed to Miniflux immediately by
elfeed tag hooks when you mark entries read/unread or add/remove the
configured star tag.  If a push fails because of a network or server
error, the entry receives =miniflux-sync-failed-tag= so you can inspect
it with =M-x miniflux-show-sync-failed= or retry the selected entries
with =M-x miniflux-retry-sync-failed=.  Retry checks the server-side
bookmark state before toggling it, because Miniflux exposes bookmark
changes as a toggle endpoint.  The next successful sync will still treat
the server as the source of truth.

=miniflux-sync-async= uses asynchronous =url.el= requests for the API
fetches, so Emacs remains responsive while Miniflux is contacted.  It
uses the same reconciliation pipeline as =miniflux-sync= and refreshes
the elfeed search buffer when possible.

* API wrapper

The package also exposes a general-purpose Miniflux API client that
you can use in your own Elisp code:

#+begin_src emacs-lisp
;; List all feeds
(miniflux-get-feeds)

;; Get entries for a specific feed
(miniflux-get-feed-entries 42 :limit 50 :status "unread")

;; Get all entries with filters
(miniflux-get-entries :limit 100 :status "unread" :starred "1")

;; Mark entries as read / unread
(miniflux-update-entry-status '(10 20 30) "read")

;; Toggle bookmark (star)
(miniflux-toggle-entry-bookmark 10)

;; Refresh feeds on the server
(miniflux-refresh-all-feeds)
(miniflux-refresh-feed 42)

;; Mark all entries in a feed or category as read
(miniflux-mark-feed-as-read 42)
(miniflux-mark-category-as-read 5)

;; List categories
(miniflux-get-categories)
#+end_src
